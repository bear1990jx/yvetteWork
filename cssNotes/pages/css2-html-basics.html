HTML常见元素和理解

head

  • meta
    <meta charset="utf-8">charset字符集, head区第一行加, 避免出现编码的问题
    <meta name="viewpoint" content="width=device-width,initial-scale=1,0,maximum-scale=1.0,user-scalable=no">视口: 手机或电脑屏幕代表的尺寸. 解读: 宽度等于设备宽度, 初始缩放为1, 做大缩放为1, 用户不能缩放. 如果不设置viewpoint, 默认宽度是980px, 在手机上会很小, 这个设置对于适配移动端非常重要.
    <base href="/">指定基础路径, 所有的链接以这个路径为基准进行计算.

HTML重要属性

  • a[href, target]
    href链接地址
    target在哪里打开, 默认是_self在当前页面打开, 可以设置为_blank在新的页面打开
  • img[scr, alt]
    scr图片地址
    alt指alternative替代资源, 图片不可用/加载失败时显示这个文字, 提示图片是什么
  • table td[colspan, rowspan]
    合并单元格
  • form[target, method, enctype]
    target表单要提交到哪里
    method用什么方式提交, 主要有get和post
    enctype指encoded type, 用什么编码, 主要针对post, 主要有两种urlencoded(提交文本)和form-data(把数据做一个编码提交, 优点是可以上传文件)
    同一个表单中name一般不同, 相同name后面的key会覆盖前面的key
    单选和复选中name相同表示是同一组, 在同一组中执行单选/复选的规则
    一个html中id不能重复
    radio单选, name相同表示是同一组, value是key的值, checked默认选中项
    checkbox多选, value是key的值, checked默认选中项
  • input[type, value]
    输入框元素
  • button[type]
    button单独使用没有效果
    submit只有出现在表单中提交才有意义, submit有button和input两种. input submit中value代表button上的文字, 比较特殊 reset只有出现在表单中提交才有意义, 重置到初始化的状态
  • select>option[value]
    下拉框, 需要select和一系列的option组合
    selected默认选中项
    multiple多选
  • label[for] 可以和表单项关联, 可用于单选框和复选框, 点击文字也可以单选或复选
    也可以将 <input> 直接放在 <label> 里, 此时则不需要 for 和 id 属性, 因为关联已隐含存在.

HTML5新增

  1. 新增区块标签:
    • section
    • article
    • nav
    • aside: 不出现在大纲中, 表示不太重要的东西, 比如广告
  2. 表单增强:
    • 日期/时间/搜索
    • 表单验证
    • Placeholder自动聚焦
  3. 新增语义:
    • header/footer头尾: 可以表示网站的头尾,也可以表示区块的头尾, 会进入页面大纲

    • em/strong强调: 替换i和b
    • i在html中保留, 用来做icon

HTML元素分类

按默认样式分:

  1. 块级block: 默认占据整行
  2. 行内inline: 和其他元素共存于一行, 选中不一定有规则形状
  3. inline-block: 对外表现和inline一样和其他元素共存于一行, 对内像block一样是一个方块有尺寸, 可以调自己的宽高, 比如select
按内容分: W3C content models

HTML元素嵌套关系

  • 块级元素可以包含行内元素
  • 块级元素不一定能包含块级元素, 比如p里不能包含div
  • 行内元素一般不能包含块级元素
    a元素可以包含块级元素

为什么a包含div合法?
查询html4.01嵌套规则, html4.01中a只能包含行内元素, 不能再包含a.
但几乎所有浏览器里都支持a包含div, html5中对这个做了合法化, 对元素分类做了更多约定, html5中a的嵌套规则中, a的内容模型是透明.
计算内容模型时, 透明元素不参与计算, 即计算嵌套合法性时, 相当于a不存在.

DIV > A
A > DIV
P > A > DIV
 p包含div不合法, 浏览器做了自己的容错修改.

如何了解一个元素嵌套是否合法?

  1. 验证工具.
  2. 查询文档中元素的content model, 被嵌套的元素是否在其中.

HTML元素默认样式和定制化

默认样式存在有必要性, 但也带来了问题, 有一些定制成本很高. 遇到奇怪的样式问题, 可以从html开始查起.
CSS Reset: 去掉默认样式, 并不是w3c的标准, 是在实践中总结的, 有非常多不同的写法. 比如, CSS Tools: Reset CSS, YUI CSS Reset.

  1. 不需要的样式统一去掉.
  2. 有一些样式各个浏览器的默认处理不同, 把需要的写一下, 让所有浏览器统一.
head里如下写法, 也可以达到reset的目的: 
            <style>
                *{
                    margin: 0;
                    padding: 0;
                }
            </style>

HTML面试真题解答

  1. doctype的意义是什么?
    • 让浏览器以标准模式渲染
    • 让浏览器知道元素的合法性, 并没有什么用, 浏览器会容错
  2. HTML XHTML HTML5的关系
    • HTML属于SGML的一个应用, SGML是一个非常通用的标记语言
    • XHTML属于XML, 是HTML进行XML严格化的结果
    • HTML5不属于SFML或XML, 比XHTML宽松
  3. HTML5有什么变化?
    • 新的语义化元素
    • 表单增强
    • 新的API(离线/新视频/图形/实时通信/本地储存/设备能力)
    • 分类和嵌套变更
  4. em和i有什么区别?
    • em是语义化标签, 表强调
    • i是纯样式的标签, 表斜体
    • HTML5中i不推荐使用, 一般用作图标
  5. 语义化的意义是什么?
    • 开发者容易理解
    • 机器容易理解结构(搜索/读屏软件)
    • 有利于SEO(search engine optimization)搜索引擎优化
    • semantic microdata在html中添加标记, 把页面中的元素作进一步标记, 方便搜索, 和语义化标签不太一样
  6. 哪些元素可以自闭合?
    • 表单元素input
    • 图片img
    • br hr
    • 源信息meta link
  7. HTML和DOM的关系
    • HTML是字符串, 是"死"的
    • DOM由HTML解析而来, 是活的
    • JS可以维护DOM
    • 源信息meta link
  8. property和attribute的区别
    • attribute属性, 写在html中的, 是"死"的. 属于html.
    • property特性, 经过解析后, 每个DOM元素会有一个property, 是活的. 属于js.
  9. form的作用
    • 直接提交表单
    • 使用submit/reset按钮
    • 便于浏览器保存表单
    • 第三方库可以整体提取值
    • 第三方库可以进行表单验证
